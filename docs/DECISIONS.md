# 설계 선택 정리

이 문서는 왜 현재 방식으로 만들었는지 쉽게 정리한 메모입니다.

## 카카오톡 발송 방식 비교

| 방식 | 비용 | 난이도 | 장점 | 단점 | 첫 버전 선택 |
| --- | --- | --- | --- | --- | --- |
| 나에게 보내기 | 무료 | 낮음 | 본인 카카오톡 `나와의 채팅`으로 바로 받기 좋음 | 다른 사람에게 보내기 어려움 | 선택 |
| 친구에게 보내기 | 무료 가능 | 중간~높음 | 특정 친구에게 보낼 수 있음 | 권한, 동의, 정책 확인이 더 필요함 | 보류 |
| 카카오톡 채널 | 무료/유료 혼합 | 중간 | 구독자에게 보내는 구조로 확장 가능 | 채널 운영 설정이 필요함 | 보류 |
| 알림톡 | 보통 유료 | 높음 | 공식 비즈니스 알림에 적합 | 비즈니스 심사와 템플릿 승인이 필요함 | 보류 |
| 텔레그램/이메일 대체 | 무료 가능 | 낮음 | 자동화가 쉬움 | 사용자가 원한 카카오톡이 아님 | 예비 대안 |

결론: 처음에는 공식 Kakao Developers의 `나에게 보내기`가 가장 단순하고 현실적입니다.

## 시장 데이터 출처 비교

| 출처 | 비용 | API 키 | 장점 | 단점 | 현재 상태 |
| --- | --- | --- | --- | --- | --- |
| Yahoo Finance chart data | 무료 | 없음 | 현재 환경에서 바로 작동, 지수/ETF 데이터를 쉽게 가져옴 | 공식 유료 API처럼 SLA가 있는 것은 아님 | 사용 중 |
| Stooq daily CSV | 무료 | 없음 | 단순 CSV 구조 | 현재 환경에서 브라우저 확인 페이지가 나와 자동화에 부적합 | 제외 |
| Alpha Vantage | 무료/유료 | 필요 | 공식 API 문서와 무료 티어가 있음 | 무료 호출 제한, 사용자가 키 발급 필요 | 후보 |
| Finnhub | 무료/유료 | 필요 | 뉴스/시장 데이터 API 제공 | 무료 호출 제한, 키 발급 필요 | 후보 |
| Polygon | 유료 중심 | 필요 | 품질과 안정성이 좋음 | 비용 부담 | 나중 후보 |

결론: 초보자가 바로 실행하는 첫 버전은 Yahoo Finance chart data로 시작하고, 안정성이 더 중요해지면 API 키 방식으로 바꿉니다.

## 뉴스 출처

뉴스는 HTML 페이지를 긁지 않고 RSS를 우선 사용합니다. RSS는 원문 기사 전문이 아니라 제목, 링크, 발행 시각 같은 공개 피드 정보를 읽는 방식입니다.

현재 기본 RSS 후보:

- Yahoo Finance RSS
- CNBC Markets RSS
- MarketWatch RSS
- Federal Reserve RSS

특정 RSS가 막히면 `.env`의 `NEWS_RSS_URLS`에서 바꿀 수 있습니다.

## 자동 실행 방식

처음 추천은 Windows 작업 스케줄러입니다.

- 무료입니다.
- 지금 쓰는 컴퓨터에서 바로 설정할 수 있습니다.
- 오전 7시 10분 실행이면 미국 정규장 마감 후라 서머타임에도 안전합니다.

완전 자동화를 원하면 나중에 GitHub Actions나 클라우드 서버로 옮기는 것이 좋습니다.

GitHub Actions 템플릿도 함께 준비해 두었습니다. 단, 카카오 토큰은 Secret으로 넣어야 하므로 처음 설정은 Windows에서 카카오 로그인을 한 뒤 진행하는 흐름이 가장 쉽습니다.

## 보고서 형태

카카오톡 메시지는 아침에 빨리 읽기 좋게 핵심만 담습니다. 대신 `reports` 폴더에는 Markdown 보고서와 HTML 보고서를 함께 저장합니다. HTML 보고서는 전체 섹터 흐름을 카드 형태로 보여주므로, 카카오톡에서 짧게 본 뒤 자세히 확인할 때 사용합니다.
